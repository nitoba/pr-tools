# Configuração

## Wizard interativo

A forma mais simples de configurar o `prt` é rodar o wizard interativo:

```bash
prt init
```

O wizard guia a configuração de:

- Providers de IA (OpenRouter, Groq, Gemini, Ollama Cloud)
- API keys com validação automática
- Azure DevOps PAT
- Reviewers padrão para criação automática de PR
- Cria `~/.config/pr-tools/pr-template.md` com o template de PR

## Arquivo de configuração

As configurações são salvas em `~/.config/pr-tools/.env`:

```bash
# Providers ativos (ordem define prioridade do fallback)
PR_PROVIDERS="openrouter,groq,gemini,ollama"

# API Keys
OPENROUTER_API_KEY="sk-or-..."
GROQ_API_KEY="gsk_..."
GEMINI_API_KEY="..."
OLLAMA_API_KEY="oa-..."

# Azure DevOps
AZURE_PAT="your-pat-token"

# Modelos (opcional — usa o modelo padrão se não definir)
# OPENROUTER_MODEL="qwen/qwen3.6-plus:free"
# GROQ_MODEL="qwen/qwen3-32b"
# GEMINI_MODEL="gemini-3.1-flash-lite-preview"
# OLLAMA_MODEL="llama3.2"

# Reviewers padrão para criação automática de PRs
# PR_REVIEWER_DEV="email@exemplo.com"
# PR_REVIEWER_SPRINT="email@exemplo.com"

# Defaults para Test Cases
# TEST_CARD_AREA_PATH="AGROTRACE\\Devops"
# TEST_CARD_ASSIGNED_TO="nome@exemplo.com"
```

<Callout type="info">
  As API keys são salvas **fora** do bloco gerenciado pelo `prt init`, portanto nunca serão sobrescritas ao reexecutar o wizard.
</Callout>

Para editar manualmente:

```bash
vi ~/.config/pr-tools/.env
```

## Precedência de configuração

Quando a mesma variável é definida em múltiplos lugares, a ordem de prioridade é:

1. **Flags CLI** — `--work-item 1234`, `--target dev`
2. **Variáveis de ambiente** — `GROQ_MODEL=qwen/qwen3-32b prt desc`
3. **Arquivo `.env`** — `~/.config/pr-tools/.env`
4. **Defaults internos** — valores padrão embutidos no binário

Exemplo: sobrescrever o modelo via variável de ambiente sem alterar o `.env`:

```bash
OPENROUTER_MODEL="qwen/qwen3-4b:free" prt desc
```

## Salvar modelo via flag

Para salvar um modelo permanentemente no `.env` sem editar o arquivo manualmente, use as flags de `prt desc`:

```bash
prt desc --set-openrouter-model qwen/qwen3-4b:free
prt desc --set-groq-model qwen/qwen3-32b
prt desc --set-gemini-model gemini-2.0-flash
prt desc --set-ollama-model llama3.2:latest
```

## Reviewers padrão

Os reviewers são usados na criação automática de PRs no Azure DevOps. Configure-os no wizard (`prt init`) ou diretamente no `.env`:

```bash
PR_REVIEWER_DEV="dev-reviewer@empresa.com"
PR_REVIEWER_SPRINT="sprint-reviewer@empresa.com"
```

## Outros arquivos

| Arquivo | Descrição |
|---------|-----------|
| `~/.config/pr-tools/.env` | API keys e configuração |
| `~/.config/pr-tools/pr-template.md` | Template da descrição de PR (editável) |
