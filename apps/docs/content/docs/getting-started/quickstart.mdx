# Quickstart

Este guia mostra o caminho mínimo: instalar, configurar e gerar sua primeira descrição de PR.

<Steps>
  <Step>
    ### Instalar

    ```bash
    curl -fsSL https://raw.githubusercontent.com/nitoba/pr-tools/main/apps/cli-go/install.sh | bash
    ```

    Após a instalação, verifique:

    ```bash
    prt --version
    ```
  </Step>

  <Step>
    ### Configurar

    Rode o wizard de configuração interativo:

    ```bash
    prt init
    ```

    O wizard vai pedir:

    1. Quais providers usar (OpenRouter, Groq, Gemini, Ollama)
    2. A API key de cada provider escolhido
    3. Seu Azure DevOps PAT (opcional — necessário para buscar work items e criar Test Cases)
    4. Emails dos reviewers padrão (opcional)

    As configurações são salvas em `~/.config/pr-tools/.env`.
  </Step>

  <Step>
    ### Gerar a descrição

    Em um repositório git, na branch da sua feature:

    ```bash
    prt desc
    ```

    O comando vai:

    1. Coletar o `git diff` e `git log` da branch atual vs base (truncado em 8000 linhas)
    2. Detectar a sprint vigente automaticamente (`sprint/NNN`)
    3. Extrair o work item do nome da branch (ex: `feat/1234-login` → work item `1234`)
    4. Buscar dados do work item no Azure DevOps (título, sprint)
    5. Enviar o contexto para o LLM com fallback automático entre providers
    6. Imprimir a descrição formatada no terminal
  </Step>
</Steps>

## Output esperado

```
 ✦ Gerando PR description...
  │ ✓  Coletando contexto git
  │    Contexto git coletado (feat/1234-login)
  │ ✓  Buscando work item #1234
  │    Work item: #1234
  │    Sprint: 42
  │    Repositório: org/projeto/repo
  │    Tentando provider: groq (qwen/qwen3-32b)...
  │ ✓  Descrição gerada (groq/qwen/qwen3-32b)
  │ └

 ✦ PR — feat/1234-login
  │ Target: sprint/42, dev
  │ Provider: groq (qwen/qwen3-32b)
  │ Work Item: #1234
```

<Callout type="info">
  Se a branch não tiver work item no nome (ex: `feat/1234-...`), a descrição é gerada sem vinculação de work item. Use `--work-item <id>` para vincular manualmente.
</Callout>
